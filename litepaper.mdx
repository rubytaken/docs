## Executive summary

heyRISA is a crypto-native AI intelligence platform that connects users to real-time market data through specialized AI agents. The platform provides institutional-grade analytics — technical analysis, sentiment tracking, wallet intelligence, and DeFi optimization — through a conversational chat interface accessible to anyone.

The platform runs 33+ domain-specific AI agents, each with access to 40+ real-data tools spanning price feeds, on-chain analytics, DEX data, and news aggregation. Users interact in natural language and receive formatted, actionable insights backed by live data from CoinGecko, DexScreener, Helius, Dune Analytics, and other providers.

heyRISA operates on a multi-model AI architecture, supporting providers including Azure OpenAI, Google Gemini, Mistral, and OpenRouter. This ensures redundancy, model diversity, and the ability to match the right model to each task.

---

## 1. Problem statement

### 1.1 Information overload in crypto markets

Cryptocurrency markets operate 24/7 across thousands of tokens, hundreds of chains, and dozens of DeFi protocols. Individual traders face an overwhelming volume of data — price movements, on-chain signals, social sentiment, token unlocks, protocol updates, and macroeconomic events — that no human can process in real time.

### 1.2 Fragmented tooling

Today's crypto trader must navigate between multiple disconnected tools:

| Tool category | Examples | Limitation |
|---------------|----------|------------|
| Price trackers | CoinGecko, CoinMarketCap | No analysis, just raw data |
| Chart platforms | TradingView | Requires manual interpretation |
| On-chain analytics | Dune, Nansen | High learning curve, SQL knowledge needed |
| Portfolio trackers | DeBank, Zapper | No AI-powered insights |
| News aggregators | CoinDesk, The Block | No sentiment scoring or impact analysis |
| DEX screeners | DexScreener | Discovery only, no strategy layer |

Each tool provides a fragment of the picture. No single platform combines all data sources with AI-powered interpretation in a conversational interface.

### 1.3 Access gap

Institutional-grade analytics — multi-indicator technical analysis, sentiment scoring, risk-adjusted portfolio optimization, and whale tracking — have historically been available only to hedge funds and proprietary trading desks. Retail traders lack access to these capabilities or the technical expertise to build them.

## 2. Solution: heyRISA platform

### 2.1 Architecture overview

heyRISA is built on a three-layer architecture:

```
┌─────────────────────────────────────────────┐
│              User Interface Layer            │
│     Chat Interface · Agent Selector ·        │
│     Model Picker · Wallet Connection         │
├─────────────────────────────────────────────┤
│            AI Agent Orchestration            │
│   33+ Specialized Agents · Tool Router ·     │
│   Multi-Model Support · Context Management   │
├─────────────────────────────────────────────┤
│            Data & Integration Layer          │
│  CoinGecko · DexScreener · Helius · Dune ·   │
│  CryptoCompare · Alternative.me · Tavily ·   │
│  Multi-Chain RPCs · Wallet Adapters          │
└─────────────────────────────────────────────┘
```

**User interface layer:** A Next.js 15 application with real-time streaming responses, dark/light mode, mobile responsiveness, and wallet connection capabilities. Users interact through natural language chat with rich formatted outputs including tables, charts, badges, and follow-up suggestions.

**AI agent orchestration:** Each agent has a domain-specific system prompt, a curated set of tools, and access to one or more AI models. The orchestration layer routes user queries to the appropriate agent and manages tool calling, context windows, and response formatting.

**Data & integration layer:** 40+ tools connect to live data providers. All data is fetched in real-time — no caching, no stale data. Tools return structured data that agents format into readable analysis.

### 2.2 Multi-model AI strategy

heyRISA does not depend on a single AI provider. The platform supports multiple models across providers:

| Provider | Models | Strength |
|----------|--------|----------|
| Azure OpenAI | GPT-4o Mini, GPT-4.1 | Reliability, tool calling, complex reasoning |
| Mistral | Pixtral Large, Mistral Large, Devstral | Multimodal analysis, coding, fast chat |
| Google | Gemini 3.1 Flash Lite, Gemma 3n | Speed, large context windows |
| OpenAI (OpenRouter) | GPT-4o Mini, GPT-OSS 120B | Fallback capacity, open-source models |
| Others (OpenRouter) | Hunter Alpha, Step 3.5 Flash | Diverse model characteristics |

This multi-model approach provides:

- **Redundancy** — if one provider experiences downtime, users switch to another
- **Task optimization** — complex analysis uses GPT-4.1, quick queries use Flash Lite
- **Cost efficiency** — free-tier models handle routine tasks, premium models handle deep analysis
- **Innovation access** — new models are added as they become available

#### Why specialization matters

General-purpose AI models struggle with crypto analysis because they lack:
- Access to real-time data (training data is stale)
- Domain-specific reasoning patterns (e.g., interpreting RSI divergence)
- Tool-calling precision (knowing which API to call for which query)

Each heyRISA agent solves these problems with a focused prompt, a curated toolset, and real-time data access.

## 6. User experience

### 6.1 Conversational interface

The chat interface is designed for crypto traders who need fast, data-rich responses:

- **Streaming responses** — results appear in real-time as the agent processes
- **Rich formatting** — tables, color-coded badges (BUY/SELL, bullish/bearish), inline charts
- **Related questions** — each response suggests follow-up queries for deeper analysis
- **Chat history** — all conversations persisted for reference and continuity

### 6.2 Agent selection

Users can either:
1. **Direct selection** — pick a specific agent from the agents page
2. **Suggestion routing** — click a pre-built suggestion that routes to the optimal agent
3. **Default chat** — type any question and the system routes to the appropriate tools

### 6.3 Crypto-native personality

heyRISA speaks the language of crypto culture. The AI uses natural degen terminology (gm, ser, fren, wagmi, ngmi, rekt, ape in) while delivering data-backed analysis. It matches user energy — bullish when the market is pumping, empathetic when positions are rekt, and contrarian when consensus is too strong.

The system detects the user's language automatically and responds in the same language, while keeping crypto slang terms in English as they are universal.

### 6.4 Time-aware interactions

The AI adjusts its personality based on the user's local time:
- **Morning (5:00-11:59)** — "gm fren" greetings
- **Afternoon (12:00-16:59)** — "hey anon" energy
- **Evening (17:00-4:59)** — "late night degen hours" vibe
